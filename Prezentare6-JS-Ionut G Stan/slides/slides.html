Watch Out!

What output generates the evaluation of the following expression?

2 + 4 + " foo " + 2 + 4
  1. "6 foo 6"
  2. "24 foo 24"
  3. "6 foo 24"
  4. "24 foo 6"
js> 2 + 4 + " foo " + 2 + 4;
"6 foo 24"

First + performs number addition, while the rest perform string concatenation.

This is a often an interview question.

Objects

  1. Create
  2. Read
  3. Update
  4. Delete

Objects: Create

Mappings from key to value, similar to Map<K, V> in Java.

var book = {}; // empty object

Keys must be strings or numbers.

var book = {
  "key": "value", // pairs are comma-delimited
  1: "value"
};

But quoting is optional when keys are valid JS identifiers.

var book = {
  key: "value",
  "illegal JS identifier": "JavaScript: the Good Parts"
};

Objects: Create

Values can be anything.

var book = {
  title: "JavaScript: the Good Parts",
  pages: 172,
  read: false,
  author: {
    firstName: "Douglas",
    lastName: "Crockford"
  } // WARNING! don't insert comma after last property
};

Objects: Read

Object properties can be read using:

  1. Bracket notation

    js> book["title"];
"JavaScript: the Good Parts"
js> book["read"];
false

  2. Dot notation (preferred)

    js> book.title;
"JavaScript: the Good Parts"
js> book.read;
false
js> book["illegal JS identifier"];
undefined

Objects: Update

Similar to reading:

  1. Bracket notation

    book["title"] = "JavaScript: the Good Parts";
book["read"]  = false;

  2. Dot notation (preferred)

    book.title = "JavaScript: the Good Parts";
book.read  = false;
book["illegal JS identifier"] = undefined;

Objects: Delete

Performed using the delete keyword.

js> var book = { title: "Title" };
undefined
js> book.title;
"Title"
js> delete book.title;
true
js> book.title;
undefined

Arrays

  1. Create
  2. Read
  3. Update
  4. Delete

Arrays: Create

An ordered sequence of heterogeneous values.

var array = []; // empty array
var array = ["foo", "bar", "baz"];
var array = ["foo", undefined, null, 1, {}];
var array = [];
array[0] = "foo";
array[1] = "bar";
array[2] = "baz";

Arrays: Read

js> var array = ["foo", undefined, null, 1, {}];
undefined
js> array.length;
5
js> array[0];
"foo"
js> array[5];
undefined
js> array.0;
SyntaxError: Unexpected number

Arrays: Update

var array = [1, 2, 3];
array[0] = "foo";
array[1] = "bar";
array[2] = "baz";
js> array;
["foo", "bar", "baz"]

Arrays: Delete

js> var array = [1, 2, 3];
undefined
js> delete array[2];
true
js> array
[1, 2, undefined]

Correct Way

js> var array = [1, 2, 3];
undefined
js> array.splice(2, 1);
[3]
js> array
[1, 2]

Equality

  • equality: == / !=
  • strict equality: === / !==
  • equality performs type coercion if operands have different types
  • strict equality will return false if types are different
  • use === unless you have really good reasons not to
  • objects, arrays, and functions are compared using reference IDs
  • all other types are compared by their values
js> 1 === 1;
true
js> "foo" === "foo";
true
js> [] === [];
false
js> ({} === {});
false
